  • Configuration ( optional )

    vkthread plugin consist of 2 files: vkthread.js and worker.js, which are locatef in the same folder. Plugin needs to know path to worker.js. At construction time plugin figures it out automatically. However, developer can set it manually. 
    vkthread.setPath("vkthread/worker.js");
  • Usage 
    vkthread.exec(fn,[arg1, ...], callback, context, dependents );
    • fn - function to execute in a thread
      mandatory 

      function foo();
	
vkthread.exec(foo, . . . );
      Function can be anonymous:
      vkthread.exec(function(n,m){return n+m;},  . . . );
    • [arg1, ...] - Array of arguments, which you want to pass along with this function
      mandatory 

      function foo(m,n){return n+m};
	
vkthread.exec(foo, [2,3], . . . );
      Array can be empty: 
      function foo(){return 'hello world'};

vkthread.exec(foo, [], . . . );
    • callback - function to process returned data. Data is passed as an argument
      mandatory 

      function foo(m,n){return n+m};
	
vkthread.exec(foo, [2,3], function(data){alert(data)} );
    • context - object, which function's this pointer will point to.
      optional

      If ommitted, thread's self is used as context. 
      function Foo(n,m){this.n = n; this.m = m;};
Foo.prototype.bar = function(){return this.n + this.m};
	
var myFoo = new Foo(2,3);
      
	
vkthread.exec(myFoo.bar, [], function(data){...}, myFoo );
    • dependencies - String; comma-separated list of files, this function depends on.
      optional 

      function Foo(n,m){this.n = n; this.m = m;};
Foo.prototype.bar = function(){return _.union(this.n + this.m) };
	
var myFoo = new Foo([1,2],[2,3]);
      	
vkthread.exec(myFoo.bar, [], function(data){...}, myFoo, 'js/underscore.js' );
      If context is ommitted, it is set at the 4-th position. 
      function bar(arr1,arr2){ return _.union(arr1,arr2) }

vkthread.exec(bar, [ [1,2],[1,3] ], function(data){...}, 'js/underscore.js' );
